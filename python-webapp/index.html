DoS a local/wireless network

Denial of Services is a common attack which flood the services with request or block users from accessing the services. Below is a demo/show case of How to DoS a network service.

First method: Deauthen using a wireless module as ESP8266 or ESP32

This is a method that use a small device as MCU module esp8266 or esp32 to discover and sending deauthenticate packets to disconnect victim from legit access point.

** Node MCU modules : the left is esp8266, another is esp32

Load code into your module. Intruction for esp32 is here , esp8266 is here

After upload code, power the module, you will get an access point "pwned" with password is "deauther" by default.

Step 2: Get to the board controller

You can either use serial comunication or web application (connect to ap created by module and go to 192.168.4.1 - default address)

Second method: Deauthen using a usb wifi adapter

This method using a wifi apdapter to discover and sending deauthenticate packets to disconnect victim from legit accespoint.

## Required : your wifi adapter should support monitor mode (for scanning) and packet injection (for sending deauth packets )
Step 1: Prepare a wireless adapter and a linux machine with aircrack-ng installed

I am using parrotKDE with aircrack-ng preinstalled.

** wifi adapter show up as wlanX (my case is wlx6......)
Step 2: Setting wireless adapter in monitor mode

You can use airmon-ng or iwconfig

airmon-ng start wlan0

or 

xxxxxxxxxx
ifconfig wlan0 down
iw dev wlan0 set type monitor 
ifconfig wlan up

** wireless adaptor now in monitor mode
Step 3 : Scan for your victim

Scan your network with airodump-ng 

xxxxxxxxxx
airodump-ng wlan0mon

You can secific your target by using 

xxxxxxxxxx
airodump-ng -d "target's BSSID" -c "target's channel number" wlan0mon
Step 4 : Deauthenticating device from network

After specific your target. Using this command to start the attack.

xxxxxxxxxx
aireplay-ng -0 0 -a "access point's BSSID" -c "device in target network's BSSID" wlan0mon

Third method : Using ARP Spoofing to redirect trafic in Local Network

This method using arpspoof to fake arp packets in order to redicrect victim packets to gateway and do not return to victim's machine so they can not access the internet.

Step 1 : Prepare a linux machine with arpspoof

I am using Kali with arpspoof preinstalled

Step 2: Discover victim ip and gateway on the network

Use ip route to get the gateway infomations. (in this case is 192.168.80.0/24)

Next you need to scan the network for victim using this command,

xxxxxxxxxx
netdiscover -r "your gateway that you get before"

Our target ip is 192.168.80.143 and having the connection to the internet as below:

Step 3: ARPspoof

Use this command to spoof target's ARP packets.

xxxxxxxxxx
arpspoof -i eth0 -t "Target's IP" -r "gateway's IP"

Now all target packet are redirected to gateway which never reach target so the connect to internet has been lost due to this attack.
